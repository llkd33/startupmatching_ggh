전문가 매칭 플랫폼 PRD (Product Requirements Document)

1. 프로젝트 개요
1.1 제품 비전
창업지원기관과 전문가를 효율적으로 연결하는 온디맨드 매칭 플랫폼 구축

1.2 핵심 목표
- 분야별 전문가 풀 구축 및 체계적 관리
- 키워드 기반 전문가 검색 및 매칭 시스템
- 요청자와 전문가 간 효율적 커뮤니케이션 채널 제공
- 창업지원기관의 니즈에 최적화된 전문가 선별 지원

2. 타겟 사용자
2.1 Primary User (요청자)
- 창업지원기관 담당자
- 창업보육센터 운영진
- 정부기관 창업지원 부서

2.2 Secondary User (전문가)
- 각 분야 전문 컨설턴트
- 강사 및 멘토
- 서비스 제공 업체
- 투자자

3. 핵심 기능 요구사항
3.1 사용자 관리 시스템
3.1.1 전문가 회원가입 및 프로필
필수 입력 정보:
- 기본 정보: 이메일, 비밀번호, 전화번호
- 핵심 이력 (간편 입력)
- 학력 정보 (간편 입력)
- 자동 생성 해시태그 확인 및 수정
- 서비스 제공 가능 지역

핵심 이력 간편 입력 시스템:
- 단계별 가이드: 회사명 → 직책 → 기간 → 주요 업무 (각 4-5개 단어 이내)
- 자동완성 기능: 기업명, 직책명 데이터베이스 기반 자동완성
- 템플릿 선택: 업종별 이력 작성 템플릿 (IT, 마케팅, 금융, 컨설팅 등)
- 원클릭 추가: + 버튼으로 이력 항목 간편 추가

학력 간편 입력:
- 학교명 자동완성 + 전공 + 졸업년도
- 주요 대학교 로고와 함께 시각적 선택

해시태그 자동 생성 시스템:
- 이력 기반 태그: 회사명, 직책, 업무 키워드에서 자동 추출
- 학력 기반 태그: 전공, 학교명에서 관련 키워드 생성
- AI 추천 태그: 유사 프로필 분석을 통한 추천 태그 제시
- 수동 추가: 사용자가 직접 추가 가능한 커스텀 태그 (최대 20개)

3.1.2 요청자 회원가입
- 기관명, 담당자 정보
- 기관 인증 시스템 (사업자등록번호 등)

3.2 캠페인(요청) 관리 시스템
3.2.1 캠페인 생성
기본 정보:
- 제목 및 상세 설명
- 카테고리 및 키워드
- 예산 범위
- 일정 및 지역
- 필요 전문가 수

첨부 파일:
- 사업계획서 등 관련 문서
- 외부 링크 연동 (Google Drive, Dropbox 등)

3.2.2 캠페인 유형별 템플릿
- 강의/멘토링 요청: 주제, 대상, 시간, 예산
- 투자자 매칭: 기업 정보, 투자 단계, 업종, 투자 규모
- 용역/업무 대행: 서비스 유형, 일정, 지역, 예산

3.3 매칭 및 알림 시스템
3.3.1 전문가 검색 알고리즘
- 키워드 매칭 점수
- 지역 기반 필터링
- 평점 및 리뷰 가중치
- 가용성 상태 확인

3.3.2 알림 발송 로직
1단계: 정밀 매칭
- 키워드 정확 일치 전문가 (상위 10-20명)
- 지역 및 가용성 고려

2단계: 확장 매칭 (1단계 응답률 30% 미만시)
- 유사 키워드 매칭
- 지역 범위 확대

3단계: 전체 공개 (2단계 후 48시간 경과시)
- 해당 카테고리 전체 전문가

3.3.3 알림 채널
- 이메일 (기본)
- 인앱 푸시 알림
- SMS (선택사항)

3.4 커뮤니케이션 시스템
3.4.1 제안서 제출
- 전문가별 맞춤 제안서 작성
- 예상 일정 및 비용 제시
- 포트폴리오 첨부

3.4.2 선별 및 선택
- 요청자의 제안서 검토 및 비교
- 전문가와의 1:1 채팅
- 최종 선택 및 계약 진행

4. 기술적 요구사항
4.1 시스템 아키텍처
- 프론트엔드: React.js 또는 Vue.js
- 백엔드: Node.js + Express 또는 Python + Django
- 데이터베이스: PostgreSQL (관계형 데이터) + Redis (캐싱)
- 파일 저장: AWS S3 또는 Google Cloud Storage
- 알림 서비스: Firebase Cloud Messaging

4.2 핵심 API 설계
4.2.1 사용자 관리
- POST /api/auth/register - 회원가입
- POST /api/auth/login - 로그인
- GET /api/users/profile - 프로필 조회
- PUT /api/users/profile - 프로필 수정

4.2.2 캠페인 관리
- POST /api/campaigns - 캠페인 생성
- GET /api/campaigns - 캠페인 목록
- GET /api/campaigns/:id - 캠페인 상세
- PUT /api/campaigns/:id - 캠페인 수정

4.2.3 매칭 시스템
- POST /api/campaigns/:id/match - 전문가 매칭 실행
- GET /api/experts/search - 전문가 검색
- POST /api/proposals - 제안서 제출

4.3 검색 및 매칭 알고리즘
```javascript
// 매칭 점수 계산 예시
function calculateMatchScore(expert, campaign) {
  let score = 0;
  
  // 키워드 매칭 (40%)
  const keywordMatch = calculateKeywordSimilarity(expert.keywords, campaign.keywords);
  score += keywordMatch * 0.4;
  
  // 지역 매칭 (20%)
  const locationMatch = calculateLocationMatch(expert.location, campaign.location);
  score += locationMatch * 0.2;
  
  // 평점 (20%)
  score += (expert.rating / 5) * 0.2;
  
  // 가용성 (20%)
  score += expert.availability ? 0.2 : 0;
  
  return score;
}
```

5. 운영 및 비즈니스 모델
5.1 수익 모델
- 매칭 수수료: 성사된 계약의 5-10%
- 프리미엄 멤버십: 전문가 우선 노출
- 광고: 관련 서비스 업체 광고

5.2 이메일 발송 비용 추정
예상 규모별 월간 비용:
- 전문가 1,000명, 월 캠페인 100개 기준
- SendGrid: 약 $15-30/월 (15,000통 기준)
- AWS SES: 약 $1-5/월 (동일 기준)
- 무료 서비스 한계: Gmail API (100통/일), Mailgun (5,000통 무료)

5.3 초기 운영 전략
- 베타 테스트: 주요 창업지원기관 3-5곳과 파일럿 진행
- 전문가 풀 구축: 각 분야별 최소 20명 이상 확보
- 품질 관리: 초기 매칭은 수동 검토 병행

6. 개발 단계별 계획
6.1 Phase 1 (MVP - 4주)
- 기본 회원가입/로그인
- 전문가 프로필 등록
- 간단한 캠페인 생성
- 수동 매칭 시스템

6.2 Phase 2 (확장 - 4주)
- 자동 매칭 알고리즘
- 이메일 알림 시스템
- 제안서 제출/관리
- 채팅 기능

6.3 Phase 3 (고도화 - 4주)
- 고급 검색 필터
- 평점/리뷰 시스템
- 결제 시스템 연동
- 관리자 대시보드

7. 성공 지표 (KPI)
7.1 핵심 지표
- 매칭 성공률: 캠페인당 평균 매칭 수
- 응답률: 알림 발송 대비 전문가 응답 비율
- 완료률: 매칭 후 실제 서비스 완료 비율
- 사용자 만족도: 요청자/전문가 평점

7.2 운영 지표
- 월간 활성 사용자 수 (MAU)
- 평균 매칭 소요 시간
- 재사용률 (리피트 고객)

8. 리스크 및 대응 방안
8.1 기술적 리스크
- 서버 부하: 클라우드 오토스케일링 적용
- 데이터 보안: 개인정보보호법 준수, SSL 인증서

8.2 비즈니스 리스크
- 전문가 확보: 초기 인센티브 프로그램
- 경쟁사 대응: 차별화된 기능과 사용자 경험 제공
