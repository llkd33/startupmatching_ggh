# Admin 기능 고도화 계획

## 📊 현재 Admin 기능 현황

### ✅ 구현된 기능

#### 1. **대시보드** (`/admin`)
- 기본 통계 카드 (사용자, 캠페인, 제안서 수)
- 최근 활동 로그
- 빠른 액션 링크
- 실시간 데이터 새로고침

#### 2. **사용자 관리** (`/admin/users`)
- 사용자 목록 조회
- 역할별 필터링 (expert/organization)
- 사용자 검색
- 사용자 초대 기능
- 프로필 정보 확인

#### 3. **캠페인 관리** (`/admin/campaigns`)
- 캠페인 목록 조회
- 상태별 필터링 (draft/active/in_progress/completed/cancelled)
- 검색 기능 (제목, 기관명, 카테고리)
- 상태 변경 기능
- 페이지네이션
- 캠페인 상세 모달

#### 4. **제안서 관리** (`/admin/proposals`)
- 제안서 목록 조회
- 상태별 필터링
- 검색 기능 (캠페인/전문가/기관)
- 페이지네이션

#### 5. **분석** (`/admin/analytics`)
- 기본 통계 (사용자, 캠페인, 제안서)
- 캠페인 유형별 분포
- 제안서 상태별 분포
- 성공 지표 (승인률, 활동률 등)

#### 6. **설정** (`/admin/settings`)
- 기본 설정 정보 표시
- Admin 사용자 생성 가이드

#### 7. **로그** (`/admin/logs`)
- Admin 활동 로그 조회
- 최근 200개 로그 표시

### 🔐 보안 기능
- Admin 인증 미들웨어 (`checkAdminAuth`)
- RLS 정책 (Admin 전용 접근)
- Admin 로그 시스템 (`admin_logs` 테이블)
- Admin 권한 부여/해제 함수

## 🛠️ 기술 스택 제안

### 차트 및 시각화
- **Recharts** 또는 **Chart.js** - 차트 라이브러리
- **React Query** - 데이터 페칭 및 캐싱
- **Zustand** 또는 **Jotai** - 상태 관리

### UI 컴포넌트
- **shadcn/ui** - 현재 사용 중 (계속 사용)
- **React Table** - 고급 테이블 기능
- **React Hook Form** - 폼 관리

### 자동화
- **Cron Jobs** - 스케줄 작업
- **Supabase Edge Functions** - 서버리스 함수
- **Queue System** - 작업 큐 관리

## 📝 참고사항

- 모든 Admin 액션은 `admin_logs` 테이블에 기록되어야 함
- 민감한 작업은 확인 다이얼로그 필요
- 대량 작업은 진행 상황 표시 필요
- 에러 발생 시 사용자 친화적 메시지 표시
- 성능 최적화를 위해 인덱싱 및 쿼리 최적화 필요
